# Appointment-Agent 本地 Baseline

## 评估口径

- 评估集：`data/eval/test.jsonl`，共 51 条；
- 36 条期望 Final，15 条期望 Tool Call；
- 主质量维度：`protocol` 与 `task_correctness`；
- Final 任务分：`70% × structured_score + 30% × reply_score`；
- Tool Call 任务分：动作、工具名和参数正确性的平均分；
- 有效通过：协议检查通过且任务正确性不低于 0.95；
- 资源指标当前为 `N/A`，性能保留原始时延和吞吐；
- 偏好先做文本归一化、精确/高置信别名匹配和否定门控，其余表达使用本地中文 Embedding `BAAI/bge-small-zh-v1.5`；余弦相似度阈值为 `0.70`，多项偏好按一对一最佳匹配计算 F1。其余评分为确定性规则，未使用外部评分 API 或 LLM-as-a-judge。

首次复现前执行 `uv sync`，再提前下载约 92 MB 的 Embedding 权重：

```powershell
uv run python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('BAAI/bge-small-zh-v1.5', device='cpu')"
```

Windows 默认缓存为 `%USERPROFILE%\.cache\huggingface\hub\models--BAAI--bge-small-zh-v1.5`。缓存完成后可设置 `$env:HF_HUB_OFFLINE="1"` 离线运行评分。

## 结论

- 0.6B 的速度优势明显，但协议与任务质量不足；
- 1.7B 的质量和速度最均衡，Tool Call 场景最佳；
- 4B 的整体质量、结构化结果、Reply、工具结果和多轮能力最高；
- 4B 的主要短板是完整信息下的工具动作决策；
- GPT-5.6-sol 在修复评测误判和字段语义后达到 51/51 有效通过，说明当前数据与消息组织可被强模型稳定完成；
- 后续应优先补强工具调用边界、工具结果继承、多轮状态和 Reply 一致性，并结合确定性工具门控与运行时优化。
