# Project Construction Log

This directory records the implementation process by spec phase.

Each phase has its own folder and `log.md`. The logs should capture:

- phase goal and deliverables;
- actual tasks completed;
- important commands and configs used;
- decisions and tradeoffs;
- problems encountered and fixes;
- produced artifacts;
- next steps.

The goal is to keep the project reproducible and reviewable, not just to store final code.
